⚙️ Scaling Action

Choose whether to add a new worker node (Scale Out) or remove an existing one (Scale In). To configure the Cluster Autoscaler, use the Machine Autoscaling Management section.

↙ Remove Worker Node

Safely removes a worker node via cordon → drain → delete. Workloads are gracefully evicted to remaining nodes before the node object is removed.

⚠ Review before proceeding
Node will be cordoned (no new pods), then all non-DaemonSet pods will be evicted to other nodes, then the node object will be permanently deleted from the cluster.
If the node is managed by a MachineSet, scale the MachineSet down instead to prevent automatic replacement.
The server and all attached volumes will be stopped and permanently deleted after the node is removed from the cluster.

○Drain — evict all pods

Automated ISO Generation

        

          Generates the agent installer ISO on your management host via SSH, writes it to a new small
          (4 GB) ISO volume using dd, then boots the worker from that ISO volume.
          The agent installer writes RHCOS to the worker's main SSD. After install the app automatically
          switches the boot device to the main SSD. Requires oc and kubeconfig on the management host.
          Management host and worker node must be in the same IONOS VDC.

🩹 Machine Health Checks

      
    

    

      A MachineHealthCheck automatically detects and remediates unhealthy nodes. When a node's
      Ready condition
      stays False or
      Unknown beyond the
      timeout, the underlying Machine is deleted so the MachineSet can provision a fresh replacement.
      The maxUnhealthy threshold acts as a safety circuit-breaker — if a large-scale
      outage takes down more than that percentage simultaneously, remediation pauses to prevent cascading deletions of
      remaining healthy infrastructure.
    

    

      
Click Refresh to load health checks from the cluster.
